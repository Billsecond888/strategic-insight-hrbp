# Step 1：调查研究（实事求是）

## 目标
深入一线，收集真实痛点，输出《业务现场真实痛点清单》。

## 核心原则
- 没有调查就没有发言权
- 拒绝二手数据，深入一线
- 让事实说话，不带预设结论

## 调研设计

### 调研方法选择
| 场景 | 推荐方法 |
|------|---------|
| 了解工作流程 | 参与式观察 + 影子跟岗 |
| 深入了解个体感受 | 半结构化访谈 |
| 收集群体共识 | 焦点小组 |
| 量化时间分配 | 工作日志分析 |

### 访谈提纲示例
1. "你工作中最让你头疼的事情是什么？"
2. "如果有一个魔法棒可以改变一件事，你会改变什么？"
3. "你觉得团队/公司最大的问题是什么？"
4. "你上次跟 Leader 沟通是什么时候？聊了什么？"
5. "你觉得公司/团队做对了什么？"

## 输出格式

```
# 业务现场真实痛点清单

## 调研信息
- 调研对象：[部门/团队]
- 调研时间：[日期]
- 调研方法：[方法]
- 样本量：[人数]

## 痛点清单
[详见 templates/痛点清单模板.md]

## 关键发现
1. [发现 1]
2. [发现 2]
3. [发现 3]

## 初步方向
1. [方向性建议]
2. [方向性建议]
```
