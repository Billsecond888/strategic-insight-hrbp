# Step 6：汇报交付

## 目标
将分析成果转化为有说服力的汇报交付物。

## 核心原则
- 故事线优先于格式
- 金字塔结构 + 毛选风格
- 先确认格式再生成

## 流程

### Step 6a：故事线设计
```
第一部分：看清形势（实事求是）
  → 基于调研的数据与事实
第二部分：抓住要害（主要矛盾）
  → 明确核心瓶颈
第三部分：依靠谁，团结谁（群众路线）
  → 干系人分析与动员
第四部分：怎么打（持久战）
  → 分阶段作战计划
```

### Step 6b：格式选择
| 格式 | 适用场景 | 调用 |
|------|---------|------|
| Markdown | 快速留档、内部共享 | 直接生成 |
| Word | 正式文档、邮件附件 | 调用 `docx` skill |
| PPT | 汇报演示 | 调用 `pptx` skill |
| HTML | 高级交互展示 | 调用 `ui-ux-pro-max` skill |

### Step 6c：生成迭代
1. 根据确认的格式生成初版
2. 展示给用户确认
3. 按反馈逐页优化

## 输出格式

```
## 交付物信息
- 格式：[Markdown/Word/PPT/HTML]
- 标题：[汇报标题]
- 受众：[谁看]
- 用途：[什么场景]

## 故事线确认
[ ] 第一部分：看清形势
[ ] 第二部分：抓住要害
[ ] 第三部分：依靠谁，团结谁
[ ] 第四部分：怎么打

## 生成状态
[ ] 初版已生成
[ ] 用户已确认
[ ] 最终版已交付
```
