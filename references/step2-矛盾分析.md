# Step 2：矛盾分析（抓主要矛盾）

## 目标
在复杂问题中找出当前阶段唯一的「主要矛盾」。

## 核心原则
- 在复杂的事物中，必有一种是主要的矛盾
- 集中力量解决主要矛盾，次要矛盾会随之缓解
- 主要矛盾是动态的，需要定期重新评估

## 分析步骤

### Step 2.1：列出所有问题
从 Step 1 的痛点清单中提取所有问题。

### Step 2.2：5-Whys 追溯根因
对每个问题追问 5 次"为什么"，直到找到根本原因。

### Step 2.3：构建因果链
画出问题之间的因果关系，找到"因"的汇聚点。

### Step 2.4：权重分析
| 问题 | 影响范围(1-5) | 严重程度(1-5) | 紧迫性(1-5) | 可改变性(1-5) | 综合得分 |
|------|:------------:|:------------:|:----------:|:------------:|:--------:|
| 问题 A | | | | | |
| 问题 B | | | | | |

### Step 2.5：锁定主要矛盾
综合得分最高且可改变性 ≥ 2 的 → 主要矛盾

## 输出格式

```
## 主要矛盾陈述
[一句话陈述]

## 判断依据
1. [依据 1]
2. [依据 2]
3. [依据 3]

## 次要矛盾清单
- [次要矛盾 1]
- [次要矛盾 2]

## 矛盾转化策略
- 当前矛盾类型：[对抗性/非对抗性]
- 转化目标：[目标类型]
- 转化方法：[具体方法]
```
