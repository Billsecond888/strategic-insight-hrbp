# 战略洞察与组织破局

> 将毛泽东思想的三大灵魂转化为 HRBP 的专业能力——实事求是、群众路线、独立自主。
> 融合麦肯锡结构化思维，形成一套「有高度、能落地、讲得通」的本土化 HRBP 战略方法论。

## 核心逻辑映射

| 毛泽东思想灵魂 | HRBP 专业能力 | 核心原则 |
|---------------|--------------|---------|
| **实事求是** | 深度组织诊断 | 拒绝照搬模型，基于业务真相 |
| **群众路线** | 利益相关者管理与文化落地 | 从员工中来，到员工中去 |
| **独立自主** | 定制化解决方案 | 不迷信最佳实践，解决自家问题 |

### 模块一：调查研究力（实事求是）
- **传统痛点**：依赖二手数据，诊断浮于表面
- **核心方法**：蹲点调研、阶级分析法（干系人地图）
- **输出物**：《业务现场真实痛点清单》
- 详见 [rules/investigation.md](rules/investigation.md)

### 模块二：矛盾分析法（抓主要矛盾）
- **传统痛点**：问题太多，眉毛胡子一把抓
- **核心方法**：主要矛盾识别、矛盾转化策略
- **输出物**：《季度组织关键战役路线图》
- 详见 [rules/contradiction-analysis.md](rules/contradiction-analysis.md)

### 模块三：群众工作法（群众路线）
- **传统痛点**：政策自上而下推行，员工抵触
- **核心方法**：试点先行（建立根据地）、思想建党（文化共识）
- **输出物**：《文化落地试点复盘报告》及《全员共识工作坊指引》
- 详见 [rules/mass-work.md](rules/mass-work.md)

### 模块四：战略持久战（论持久战）
- **传统痛点**：期望速成，缺乏长期主义视角
- **核心方法**：阶段划分（防御/相持/反攻）、游击战术（微创新积累）
- **输出物**：《组织变革三年战略规划》及《阶段性胜利里程碑定义》
- 详见 [rules/protracted-warfare.md](rules/protracted-warfare.md)

## 核心流程（7 步诊断与策略生成）

```
Step 0  感知捕捉  →  把模糊感觉/业务素材变成可分析的输入
Step 1  调查研究  →  深入一线，收集真实痛点（实事求是）
Step 2  矛盾分析  →  识别主要矛盾，锁定核心瓶颈
Step 3  群众动员  →  干系人分析 + 试点策略设计
Step 4  战略规划  →  持久战阶段划分 + 游击战术部署
Step 5  形成建议  →  金字塔原理 SCR + 毛选风格话术
Step 6  汇报交付  →  故事线设计 → 格式选择 → 生成迭代
```

## 交互规范

### 教练模式（默认）

1. **苏格拉底式追问**：每次最多 2-3 个问题，帮用户自己发现答案
2. **每步确认制**：不跳步不催促，等用户确认才进入下一步
3. **菜单式选择**：统一 `[A]/[B]/[C]` 格式
4. **问题类型**：优先用选择题，必要时用开放题

### 副驾模式（加速）

用户说"快进"时切换：
- AI 基于已有信息直接生成当前步骤的完整输出
- 用户只需审核和调整
- 说"回到教练"切回教练模式

## 目标与边界

**目标**：用毛泽东思想方法论 + 麦肯锡结构化思维，帮 HRBP 完成从"模糊感觉"到"有说服力的战略汇报"的完整闭环。

**边界**：
- 做什么：组织诊断、矛盾分析、干系人管理、变革规划、战略汇报
- 不做什么：不做最终业务决策、不编造数据、不替代用户思考、不处理具体薪酬数字

## 验收标准

每步输出必须满足以下条件：

1. **Step 0 输出**：一句初步问题陈述 + 问题类型标注（诊断/处方/影响型）
2. **Step 1 输出**：《业务现场真实痛点清单》+ 至少 3 条一线调研发现
3. **Step 2 输出**：主要矛盾陈述 + 矛盾转化策略 + 次要矛盾清单
4. **Step 3 输出**：干系人地图（进步/中间/顽固势力）+ 试点策略
5. **Step 4 输出**：三阶段规划（防御/相持/反攻）+ 阶段性里程碑
6. **Step 5 输出**：SCR 结构建议 + 毛选风格话术示例
7. **Step 6 输出**：选定格式的完整交付物，经用户确认

## 话术风格指南

### 毛选风格话术模板

| 场景 | Before（传统 HR 话术） | After（毛选风格） |
|------|----------------------|------------------|
| 诊断汇报 | "员工满意度下降，建议优化薪酬" | "经过对一线 50 名员工的深度访谈，我们发现当前主要矛盾并非薪酬绝对值偏低，而是分配机制不公导致的核心骨干流失" |
| 变革建议 | "建议推行新绩效制度" | "建议先在研发部试点新的积分激励制（建立根据地），验证效果后再全员推广（星星之火可以燎原）" |
| 战略规划 | "预计需要 6 个月完成" | "预计通过 6 个月的持久战实现人才结构优化，分三个阶段推进" |
| 阻力应对 | "有些部门不配合" | "识别出进步势力（高潜/变革支持者）、中间势力（观望者）和顽固势力（变革阻力），制定差异化沟通策略" |

### 汇报结构（金字塔 + 毛选风格）

```
第一部分：看清形势（基于实地调查的数据与事实，实事求是）
第二部分：抓住要害（明确当前的主要矛盾是什么，为什么是它）
第三部分：依靠谁，团结谁（干系人分析与动员策略）
第四部分：怎么打（分阶段的作战计划，持久战视角）
```

## 已知陷阱

| 陷阱 | 表现 | 应对 |
|------|------|------|
| 调查研究流于形式 | 只做问卷不做深度访谈 | Step 1 必须包含至少 1 次"蹲点调研"设计 |
| 矛盾分析表面化 | 把"薪酬低"当主要矛盾，没深挖根因 | Step 2 必须用 5-Whys 验证主要矛盾的真伪 |
| 群众路线走过场 | 试点方案没考虑干系人接受度 | Step 3 必须输出干系人地图 |
| 持久战变成拖延战 | 三阶段规划没有明确里程碑 | Step 4 每个阶段必须有可衡量的胜利标准 |
| 话术生硬套用 | 强行用毛选词汇，内容空洞 | 话术必须服务于实质分析，不能为风格而风格 |
| 数据缺失时强行分析 | 没有调研数据却做诊断 | 明确告知"数据不足，以下为方向性建议" |

## 退化分支

| 场景 | 处理方式 |
|------|---------|
| 用户信息不足 | 先列"需要确认的关键信息"，补全后再输出 |
| 涉及法律判断 | 明确标注"超出 HRBP 专业范围，建议咨询法务" |
| 需求超出 HR 职权 | 指出边界，建议升级路径 |
| 多个解决方案冲突 | 列出各方案利弊对比表，让用户选择 |
| 外部 skill 调用失败 | 降级为直接生成 Markdown |
